# 成为一个可以构建大型可靠分布式系统的架构师

这里讲述 Java 程序员成长为架构师的路径

#日记训练/2026/新知

今天在对自己的知识复盘的时候，我用了一句话去总领我的知识。就是：成为可以构建可靠的大型分布式系统的架构师。 当我写完这句话的时候，我瞬间一惊。原来之前模模糊糊一直学习的东西就是为了成为这样的一个角色，或者说是这样的一个人物。但我现在对于构建可靠的大型分布式系统的架构师的水平还是处于初级的水平。我对于自己的期待应该是专家级别的。 首先我对成为这样的人物内心还是向往的。 但是我在进入程序员这个行业的时候，自己并不知道自己是要成为这么一个角色，有的话，也是比较模模糊糊的。
为什么我突发对这点有所感悟，因为我瞬间悟到了。一个人的能力高低，不是靠着按部就班地学习知识，能力就会慢慢变强的。 一个人的能力变强一定是跟他做的事情有关，做的事情的能力在不断增强，在其中锻炼出来的能力也就会慢慢变强。 而一个人会选择做什么事，又跟这个人立了什么志向，或是弄清楚了自己要成为一个什么样的密切相关。 所以，我想，当时我要是有要一个非常明确的目标，成为一个可以构建可靠的大型分布式系统的架构师， 然后我慢慢去拆解要成为这样的架构师要做什么事情，如何一步一步成为一个专家级别的架构师，或许我现在就是要给专家级的架构师了。 或许也可能是更早。 还有一点，我自己回忆起来，就是我觉得是自己不够自信，不相信自己有能力成为专家级别的架构师。 因为，我从来就不敢想自己可以去大厂，或者说是有能力去大厂。 所以，总结一下，第一不清楚自己想要成为什么样的人，第二是不自信，不相信自己想清楚了要成为什么样的人之后，只要自己努力，就一定能够成为这样的 人。
在这里顺便，我也想说说，之所以会专业，是思维惯性的问题，这个思维惯性跟教育有关， 就是我们动一年级读书直接到大学， 这个过程就是自己不断努力积累知识，在这个过程中，自己确实就是按部就班地学习知识，然后能力慢慢变强的。 然后你大学毕业了之后， 你也因为自己只要自己按部就班的努力，就可以成为一个厉害的人，其实没有这个逻辑。 我们在受教育阶段，按部就班的能力变强，前提是背后有整个教育部在给你定锚，就是她给你定了个目标，考上大学。成为一个大学生。 因为你自己没有参与目标的制定，你就忽略了你所有的努力是因为有这个目标在牵引的。 你出了社会，这个时候没有人给你定目标了，而且你自己本身也没有锻炼过这样的能力，就是成为什么的人，这个我们受教育的时候，从来没有被训练过。
所以，变成很厉害的人，不是按部就班就能来的，而是你要给自己定个目标，成为什么样的人。然后，你就会去拆解成为这样的人需要什么能力，没有你自然就会去锻炼这方面的能力。 能力是在做事中练出来的， 你要自己有多大的能力，你就给自己立多大的目标。目标要立的足够大，你的能力才有想象空间。
